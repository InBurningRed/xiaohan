## Summary

在现有纯静态番茄钟 demo（pomodoro-demo/）基础上增强：

- 圆环中间稳定显示数字倒计时（mm:ss），不再依赖负 margin
- 增加“自主调整时间”设置：专注时长 + 强制休息时长（默认 25/5 分钟），持久化到 localStorage
- 专注结束后进入“强制休息”阶段：自动开始倒计时、允许暂停/继续、禁止跳过与模式切换；强制休息结束后回到专注（需手动开始）
- 保留“每 4 个专注后长休”的逻辑：第 4 个专注结束 → 强制休息结束后进入长休（需手动开始）
- 每个阶段结束播放提示音（沿用现有 WebAudio beep）

## Proposed Changes

### 1) 倒计时居中显示（HTML/CSS 结构调整）

目标：保证倒计时数字始终位于圆环几何中心，兼容响应式，不依赖负 margin。

文件与改动：

- [index.html](file:///d:/xiaohan/github_try/xiaohan/pomodoro-demo/index.html)
  - 将 `.time` 与 `.hint` 放入 `.ring` 内部（新增 `.ring-content` 容器），结构从：
    - `.ring` + `.time` + `.hint`（兄弟）
    - 调整为：`.ring` 内部包含 `::before` 内圆 + `.ring-content`（内容层）
- [styles.css](file:///d:/xiaohan/github_try/xiaohan/pomodoro-demo/styles.css)
  - 移除 `.time/.hint` 的负 margin 方案
  - 使用 `.ring { position: relative; }` + `::before` 作为内圆层 + `.ring-content { position: relative; z-index:1; display:grid; place-items:center }`
  - 为数字添加 `font-variant-numeric: tabular-nums`，避免数字宽度变化带来抖动
  - 保持现有 `conic-gradient` 进度环方案不变

### 2) “自主调整时间”设置（专注 + 强制休息，默认 25/5）

目标：允许用户调整专注时长与强制休息时长；保存后立即生效并持久化。

UI 方案：在 header 增加“时长”按钮，弹出原生 `<dialog>` 设置面板。

文件与改动：

- [index.html](file:///d:/xiaohan/github_try/xiaohan/pomodoro-demo/index.html)
  - header 区新增按钮：`data-action="settings"`（与“通知”同样式）
  - body 末尾新增 `<dialog>`（两项设置 + 按钮：取消 / 恢复默认 / 保存）
    - 专注分钟数：`focusMinutes`
    - 强制休息分钟数：`forcedBreakMinutes`
- [styles.css](file:///d:/xiaohan/github_try/xiaohan/pomodoro-demo/styles.css)
  - 补充 dialog 的基础样式（遮罩、面板、表单行、按钮区），沿用现有圆角/边框/阴影体系
- [app.js](file:///d:/xiaohan/github_try/xiaohan/pomodoro-demo/app.js)
  - 把 `MODES` 拆为“默认值 + 可覆盖配置”：
    - `DEFAULTS = { focusSec, forcedBreakSec, longBreakSec }`
    - `config = loadConfigFromLocalStorage(DEFAULTS)`
  - 保存时写入 localStorage（例如 key：`pomodoroConfigV1`）
  - 应用策略（保存时）：
    - 若当前不在运行：直接刷新当前 mode 的 `totalSec/remainingSec`
    - 若当前在运行：按“已消耗时间不变”的规则调整
      - `elapsed = totalSec - remainingSec`
      - `totalSec = newTotal`
      - `remainingSec = max(0, totalSec - elapsed)`
      - `endAtMs = Date.now() + remainingSec * 1000`

说明：

- 长休时长默认保持 15 分钟（本次不提供 UI 调整），因为用户选择“专注+休息”而非“三种都可调”。

### 3) 专注结束后强制休息（自动开始，允许暂停，禁用跳过/切换）

目标：专注结束后进入“强制休息”阶段，强制休息不可跳过/不可切模式；允许暂停/继续；结束后回到专注（不自动开始）。

状态/字段设计（最小侵入）：

- 继续沿用 `mode`（focus/short/long），新增锁字段：
  - `lock = null | 'forced_break'`
  - `pendingLong = boolean`（用于在强制休息结束后决定是否进入长休）

行为细则（按用户选择落地为唯一规则）：

- 专注完成（自然到 0 或点“跳过”）：
  - 记录本次专注完成后的轮次计数（保持现有 round/todayFocusMin 统计）
  - 设置 `pendingLong = (focusRound === 4)`（以现有 round 语义为准，执行时会以代码实际逻辑对齐）
  - 进入强制休息：
    - `lock = 'forced_break'`
    - `mode = 'short'`（强制休息借用 short 主题色/展示，也可改为独立展示文案）
    - `totalSec = remainingSec = config.forcedBreakSec`
    - 立即 `startTimer()` 自动开始
    - toast/通知文案：`专注结束，强制休息 mm:ss`
- 强制休息中：
  - 允许：开始/暂停（toggle）
  - 允许：重置（reset 恢复到 forcedBreakSec）
  - 禁止：跳过（skip），禁用按钮并在 `skip()` 入口 guard
  - 禁止：切换 tab（已有 running guard；额外加 `lock` guard，并做 UI disabled 态）
  - hint 文案显示：`强制休息中…`
- 强制休息完成：
  - `lock = null`
  - 若 `pendingLong=true`：切换到 `mode='long'`，加载 `DEFAULTS.longBreakSec`，不自动开始
  - 否则：切换到 `mode='focus'`，加载 `config.focusSec`，不自动开始
  - `pendingLong=false`
  - 播放提示音 + toast/通知：`休息结束` 或 `进入长休`

### 4) 保留 4 次专注后长休（在强制休息之后）

目标：不改变用户“强制休息结束后需手动开始”的偏好，同时保留长休阶段。

规则：

- 第 4 次专注完成后仍先进入强制休息
- 强制休息完成后进入长休（15 分钟，需手动开始）
- 长休完成后回到专注（需手动开始），并将番茄轮次重置为 1

### 5) 键盘快捷键与按钮禁用一致性

文件与改动：

- [app.js](file:///d:/xiaohan/github_try/xiaohan/pomodoro-demo/app.js)
  - 在键盘事件（Space/R）与按钮事件中统一检查 `lock === 'forced_break'`：
    - Space：允许（用于暂停/继续强制休息）
    - R：允许（重置强制休息）
    - Skip：禁止（按钮禁用 + 入口 guard）
    - Tabs：禁止（入口 guard + UI disabled）
- [styles.css](file:///d:/xiaohan/github_try/xiaohan/pomodoro-demo/styles.css)
  - 为 disabled 按钮增加样式（透明度、cursor、hover 无效），避免“能点但没反应”的困惑

## Assumptions & Decisions

- “中间显示数字倒计时”理解为：圆环中心稳定显示 `MM:SS`（现有格式），并修复当前因负 margin 导致的错位/空白问题。
- “强制 5 分钟休息”按最终确认改为：强制休息不可跳过，但分钟数可在设置中调整（默认 5）。
- 强制休息自动开始；允许暂停/继续与重置；不允许跳过与切模式。
- 强制休息结束后回到专注（需手动开始）；仅当第 4 个专注完成时，强制休息结束后进入长休（需手动开始）。
- 长休时长保持 15 分钟（本次不做 UI 调整）。
